# Git は「セーブポイント」をつくる道具

コードを書いていると「昨日までは動いていたのに」が必ず起きます。Git はそのための保険です。

- **リポジトリ** — 履歴を保存する箱。`git init` でフォルダに 1 つつくります
- **コミット** — セーブポイント。「いつ・何を・なぜ変えたか」の記録
- **ステージ** — 次のコミットに入れるものを選ぶ待機場所

```
作業フォルダ ── git add ──▶ ステージ ── git commit ──▶ 履歴
```

<Callout type="info">
「全部の変更」ではなく「選んだ変更」を記録できるように、間にステージという待機場所が挟まっています。
</Callout>
